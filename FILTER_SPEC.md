# 写真一覧フィルタ仕様

- 作成日: 2026-07-21
- 最終更新日: 2026-07-22
- 対象バージョン: v1
- 状態: 第1段階（型付きモデル・評価器・コア単体テスト）と第2段階（表示分離・選択・件数表示）実装済み。第3段階は未実装

## 1. 目的

スキャン結果の一覧を、ファイル種別、ファイルシステム情報、Exif情報、取込状態などの条件で絞り込めるようにする。

フィルタによる表示状態とコピー対象のチェック状態は別の状態として管理する。通常は絞り込みで非表示になった項目をコピー対象から外すが、多量のファイルについてチェック済みの項目を複数の絞り込み条件で確認できるよう、表示外になってもチェックを維持するモードを提供する。

フィルタ項目には [TEMPLATE_SPEC.md](TEMPLATE_SPEC.md) のトークン名を再利用するが、テンプレートの書式化結果を文字列として比較しない。文字列、数値、日時、真偽値などの元の型付き値を比較する。

## 2. 対象項目

### 2.1 ファイル分類・状態

次の項目をフィルタ対象とする。

- ファイル種別
- 拡張子
- コピー計画の状態: 未取込、上書き対象、取込済、競合、スキャンエラー、コピーエラー
- Exif読込状態: 読込済み、Exif情報なし、未対応形式、読取エラー

ファイル種別は拡張子から判定し、ファイル内容のシグネチャ判定は行わない。拡張子は先頭の `.` を含む小文字へ正規化し、大文字・小文字を区別せず照合する。初期分類は次のとおりとし、拡張子集合はコアで一元管理する。

| ファイル種別 | 拡張子 |
|---|---|
| JPEG | `.jpg` `.jpeg` |
| RAW | `.arw` `.cr2` `.cr3` `.dng` `.nef` `.nrw` `.orf` `.raf` `.rw2` `.pef` `.srw` |
| その他の画像 | `.bmp` `.gif` `.heic` `.heif` `.png` `.tif` `.tiff` `.webp` |
| 動画 | `.avi` `.m4v` `.mov` `.mp4` |
| その他 | 上記に属さない拡張子、または拡張子なし |

ファイル種別と拡張子は別の条件とする。たとえばファイル種別でRAW全体を選択したうえで、拡張子条件により `.arw` だけへ絞り込める。

### 2.2 ファイルシステム系トークン

次のトークンを対象とする。

- 文字列: `{OriginalName}` `{FileName}` `{Extension}` `{SourceRelativeDirectory}`
- 日時: `{ModifiedDate}`
- 数値: `{FileSize}`
- 真偽値: `{Protected}`
- 連番: `{Sequence}`

`{SourceRelativeDirectory}` はコピー元ルート直下の場合も空文字という既知の値とし、Unknownにはしない。`{Sequence}` が付与されていない場合はUnknownではなく「連番なし」という既知の状態とする。

### 2.3 Exif系トークン

次のトークンを対象とする。

- 日時: `{TakenDate}` `{TakenDateLocal}` `{TakenDateInTimeZone}`
- 文字列: `{CameraMake}` `{CameraModel}` `{CameraSerial}` `{Lens}`
- 整数: `{Width}` `{Height}` `{ExifWidth}` `{ExifHeight}` `{Orientation}` `{Iso}` `{FocalLength35mm}`
- 小数・有理数: `{Aperture}` `{ShutterSpeed}` `{ExposureTime}` `{FocalLength}` `{GpsLatitude}` `{GpsLongitude}` `{GpsAltitude}`
- 評価: `{Rating}`
- 真偽値: `{HasGps}`

比較には書式化前の値を使う。単位を表す `F`、`mm`、`m`、`s` や、テンプレート用にサニタイズされた文字は比較値へ含めない。

- `{Width}` と `{Height}` はOrientationを反映した寸法を比較する
- `{ShutterSpeed}` と `{ExposureTime}` はともに露光秒数へ変換して数値比較する
- `{Rating}` の1～5は数値範囲で比較できる。`Rejected` は数値 `-1` をUIへ見せず独立した選択肢とする
- `{HasGps}` は `GPS` / `NoGPS` の2値とし、Unknownを持たない

撮影日時フィルタでは、Exifの撮影日時が欠落している場合にファイル更新日時へフォールバックせずUnknownとする。これにより「Exif撮影日時が存在しないファイル」を抽出できるようにする。

- `{TakenDate}` はExifに記録された壁時計値を比較する
- `{TakenDateLocal}` は有効なExifオフセットがあればPCの現在のタイムゾーンへ変換する。オフセットがない、または不正な場合は記録された壁時計値を比較する
- `{TakenDateInTimeZone}` は条件で指定したタイムゾーンへ変換する。有効なExifオフセットがない場合は記録された壁時計値を比較する

## 3. 条件と結合規則

### 3.1 条件集合

- 条件がない場合は全項目を表示する
- 複数の条件はANDで結合する
- ファイル種別、拡張子、状態など、一つの条件内で複数の選択肢を指定した場合はORで結合する
- v1では任意のORグループや条件の入れ子は提供しない
- スキャンエラー行とコピーエラー行も通常の一覧項目として評価し、必要な値を取得できないフィールドについてはUnknownとして扱う。エラー行を明示的に抽出するため、コピー計画の状態条件を提供する

### 3.2 文字列条件

文字列項目では次の比較方法を提供する。

- 完全一致
- 部分一致
- ワイルドカード
- 正規表現

各条件には「一致する項目を対象にする」または「一致する項目を対象から外す」を指定する。

既定では大文字・小文字を区別しない。必要な場合は条件単位で区別を有効にできる。大文字・小文字を区別しない比較には現在のUIカルチャではなく、カルチャに依存しない比較を使う。

ワイルドカードは次の規則とする。

- `*`: 0文字以上の任意の文字列
- `?`: 任意の1文字
- パターンはフィールド全体へ一致させる

正規表現は.NETの正規表現として解釈し、`Regex.IsMatch` と同じ部分一致規則を使う。全体一致が必要な場合は利用者が `^` と `$` を指定する。無効な正規表現を含む条件は適用できない。正規表現評価にはタイムアウトを設定し、タイムアウトした条件はエラーとして通知する。動的なパターンへ `RegexOptions.Compiled` は使用しない。

### 3.3 数値条件

数値項目では最小値と最大値をそれぞれ省略可能とし、指定された境界値を含める。

```text
最小値 <= 値 <= 最大値
```

両方が未指定の条件は無効とする。最小値が最大値を上回る条件は適用できない。

`{FileSize}` の入力には B、KiB、MiB、GiB の単位を指定できる。内部ではバイトへ変換し、`1 KiB = 1024 B` とする。Exif数値は書式化や表示単位を適用する前の値を比較する。

### 3.4 日時条件

日時項目では開始と終了をそれぞれ省略可能とし、指定された境界値を含める。

- 日付だけを指定した開始値は、その日の00:00:00を含む
- 日付だけを指定した終了値は、その日の翌日00:00:00未満として評価し、終了日全体を含む
- 時刻まで指定した場合は指定時刻を境界値として扱う
- `{ModifiedDate}` はスキャン時に取得したPCローカル時刻を比較する
- 撮影日時の変換規則は §2.3 に従う

UIには比較対象のタイムゾーンまたは「Exif記録値」であることを表示する。

### 3.5 真偽値・列挙条件

真偽値と列挙値はチェックリストまたは単一選択で指定する。選択肢が一つもない条件は無効とする。

## 4. Unknown

### 4.1 定義

Unknownは文字列リテラル `"Unknown"` ではなく、対象フィールドに比較可能な型付き値が存在しない状態とする。実データに文字列 `"Unknown"` が記録されている場合は既知の文字列として扱う。

Exif系フィールドでは次の場合がUnknownになり得る。内部では理由を保持し、UI上は必要に応じてExif読込状態条件で区別できるようにする。

- 対象Exif項目の欠落
- `NoMetadata`
- `Unsupported`
- `ReadError`

Exifをまだ読み込んでいない状態はUnknownではない。Exif条件を適用する場合は §5 の規則でExifを読み込んでから条件を評価する。

値が必ず存在する拡張子、`{Protected}`、`{HasGps}` などではUnknown指定を無効表示する。ただしスキャンエラーによりファイル情報自体を取得できない行はUnknownになり得る。

### 4.2 対象・除外との組合せ

各条件について、既知の値に対する一致動作とは独立して「Unknownを含める」または「Unknownを除外する」を指定する。

| 条件の動作 | Unknown | 条件を満たす項目 |
|---|---|---|
| 一致する項目を対象にする | 含める | 一致する既知値、またはUnknown |
| 一致する項目を対象にする | 除外する | 一致する既知値のみ |
| 一致する項目を対象から外す | 含める | 一致しない既知値、またはUnknown |
| 一致する項目を対象から外す | 除外する | 一致しない既知値のみ |

Unknownを持ち得るフィールドの既定値は「Unknownを除外する」とする。

## 5. Exif読込とフィルタ適用

適用する条件集合がExif系フィールドまたはExif読込状態を使用する場合、条件集合は `RequiresExif = true` とする。

- テンプレートがExifを使わず、設定「テンプレートで未使用の場合もExif情報を読み込む」がOFFでも、有効なExifフィルタがあればExifを読み込む
- 既存の一覧に必要なExif情報がない状態で「適用」を押した場合は、Exifスキャンを開始し、既存の進捗表示とキャンセル操作を使用する
- Exifスキャンが完了してから新しい条件を有効にする
- Exifスキャンをキャンセルまたは全体エラーで中止した場合は、直前の有効な条件と表示結果を維持する。作成途中の表示結果へ切り替えない
- `ReadError` になった項目は一覧に残し、Exif系フィールド値をUnknownとして評価できるようにする
- RAW+JPEGペアのExif値は既存の解析元決定規則に従う。ファイルシステム系の値は常に評価対象ファイル自身の値を使う

## 6. 適用操作とUI

フィルタ編集は一覧上部の折りたたみ可能な非モーダル領域で行う。条件行には少なくとも次を表示する。

- 対象項目
- 比較方法
- 型に応じた値入力
- 対象にする／対象から外す
- Unknownを含める／除外する（対象フィールドで利用可能な場合）
- 条件の削除

「条件を追加」「適用」「クリア」を提供する。入力中の条件は編集中状態とし、文字入力ごとの自動適用は行わない。チェック解除を伴う場合があるため、表示と選択状態を変更するのは「適用」を実行した時点とする。

- 無効な条件がある間は「適用」を無効にし、行内に理由を表示する
- 「クリア」は条件を空にして全項目を表示するが、既に外れたチェックを復元しない
- 適用済み条件数を一覧上部へ表示する
- v1では条件と選択解除設定を永続化せず、アプリ起動時は条件なし、§7.1の既定動作で開始する

## 7. 表示状態とチェック状態

### 7.1 非表示項目のチェック

次の設定を一覧またはフィルタ領域の近くへ常時表示する。

```text
絞り込みで非表示になった項目のチェックを外す
```

- 既定はONとする
- ONの場合、フィルタの適用に成功した時点で、非表示となったコピー可能項目のチェックを外す
- OFFの場合、非表示になってもチェック状態を変更しない
- OFFからONへ切り替えた時点でフィルタが有効なら、現在表示外のコピー可能項目のチェックを外す
- フィルタを解除しても、フィルタによって外れたチェックを自動復元しない
- アプリ再起動時はONへ戻す
- 取込済、競合、スキャンエラー、コピーエラーなどコピー不能な項目は従来どおり常に未チェックとする

### 7.2 全選択

一覧ヘッダーの3状態チェックボックスは、現在表示中かつコピー可能な項目だけを対象とする。

- 表示中のコピー可能項目がすべて選択済みならチェック
- 一部だけ選択済みなら中間状態
- 選択なし、または表示中にコピー可能項目がなければ未チェック
- クリック時、全選択状態なら表示中の項目を全解除し、それ以外なら表示中の項目を全選択する
- 表示外項目のチェック状態は全選択操作で変更しない

### 7.3 コピー対象

コピーは表示状態ではなく、全スキャン結果に含まれるチェック済みかつコピー可能な項目を対象とする。チェック維持モードでは表示外のチェック済み項目もコピーされる。

誤認を防ぐため、一覧付近に少なくとも次の件数を表示する。

```text
表示 120 / 全 3,842
チェック 1,260（表示外 1,180）
```

コピーボタンまたはその近くにも総コピー対象件数を表示する。表示外のチェック済み項目がある場合は、その件数を常に可視にする。

一覧で選択中の詳細表示対象がフィルタによって非表示になった場合は、詳細選択を解除する。自動的に別の行を選択しない。

### 7.4 サマリー

既存の未取込、上書き対象、取込済、競合・エラーの件数は全スキャン結果を基準とする。これに加えて、表示件数、全件数、チェック総数、表示外チェック数を区別して表示する。

## 8. 実装方針

全スキャン結果と表示用コレクションを分離する。

```text
ObservableCollection<PreviewItem> Items
    └─ ICollectionView ItemsView
         └─ DataGrid.ItemsSource
```

- `Items` は全スキャン結果とチェック状態を保持する
- `ItemsView` は適用済み条件による表示だけを担当する
- コピー、チェック総数、全体状態集計は `Items` を使う
- 全選択と表示件数は `ItemsView` を使う
- フィルタ評価器はUIから分離し、型付きの候補値に対する単体テストを可能にする
- フィルタ項目定義には値の型、Exif要否、Unknown可否を保持する
- 正規表現などの入力は「適用」時に一度検証・準備し、項目ごとに再構築しない

数千件規模ではWPFの `ICollectionView.Filter` と既存の行仮想化を使用する。実測でUI停止が問題になる規模では、適用済み条件から一致項目集合をバックグラウンドで計算し、UIスレッドでは表示更新だけを行う。バックグラウンド評価中に新しい適用要求が来た場合は古い評価結果を破棄する。

### 8.1 実装段階

一覧フィルタは次の3段階に分け、各段階で単体テストまたはUI連携テストを通してから次へ進む。

1. **型付きモデル・評価器・コア単体テスト**
   - フィルタ項目定義、型付き候補値、Known／Unknown／Exif未読の区別を実装する
   - 文字列、数値、日時、真偽値、列挙条件の検証と評価を実装する
   - AND／条件内OR、対象／除外、Unknown、正規表現タイムアウトをコア単体テストで固定する
2. **表示分離・選択・件数表示**
   - 全スキャン結果の `Items` と表示用 `ItemsView` を分離する
   - 表示中だけの全選択、非表示チェックの解除／維持、コピー対象と各件数の集計を実装する
   - 再スキャンとコピー後の選択復元がフィルタ表示と独立して動作することを確認する
3. **WPFフィルタUI・Exif読込連携**
   - 編集中条件と適用済み条件を分離した非モーダルUIを実装する
   - Exif未読時は既存一覧と直前の適用済み条件を保持したままExifを読み、成功後だけ新条件を反映する
   - キャンセル、読取エラー、RAW+JPEG解析元共有を含むUI連携を確認する

### 8.2 現在の実装状況

2026-07-22 時点で第1段階と第2段階を実装済み。

- `PhotoImporter.Core.Filtering` に型付き項目定義、フィルタ候補、条件モデル、事前検証、準備済み評価器を追加
- 完全一致、部分一致、ワイルドカード、正規表現、大小文字区別、正規表現タイムアウトに対応

- 複数条件AND、選択肢OR、対象／除外とUnknownの組合せを実装
- `{Sequence}` は「連番なし」と数値範囲、`{Rating}` は1～5の数値範囲と `Rejected` を同一条件内でOR指定可能とした
- 撮影日時は更新日時へフォールバックせず、Orientation反映寸法と露光秒数を型付き値として評価する
- ファイル分類をコアで一元化し、既存のRAW+JPEG解析も同じJPEG／RAW拡張子定義を使用する
- テンプレートとフィルタのOrientation反映寸法計算を共通化した
- 全スキャン結果の `Items` と `ICollectionView` の `ItemsView` を分離し、フィルタ適用後も全件とチェック状態を保持
- 全選択は表示中のコピー可能項目だけを対象とし、コピー対象は表示状態にかかわらず全件から集計

- 表示件数、全件数、チェック総数、表示外チェック数を常時表示し、コピーボタンにも総コピー対象件数を表示
- フィルタ解除でチェックを復元せず、再スキャンとコピー後の選択復元後にも適用済み表示と非表示チェック方針を再適用
- Release構成で `dotnet test PhotoImporter.sln -c Release --no-restore` を実行し、全217件成功

第3段階のWPFフィルタ編集UIと遅延Exif読込連携は未実装。

## 9. 受入条件

少なくとも次を自動テストまたはUIテストで確認する。

- 文字列の完全一致、部分一致、ワイルドカード、正規表現
- 大文字・小文字を区別する場合としない場合
- 無効な正規表現と正規表現タイムアウト
- 数値・日時の片側範囲、両側範囲、境界値
- 日付だけを指定した終了日に、その日のデータが含まれること
- 複数条件のANDと、一条件内の複数選択肢のOR
- 対象にする／対象から外すとUnknownの4組合せ
- 実データの文字列 `"Unknown"` と欠落値を区別すること
- 撮影日時欠落時に更新日時へフォールバックせずUnknownになること
- Exif条件を適用した場合だけ必要なExifスキャンが開始されること
- Exifスキャンのキャンセル時に直前のフィルタ結果を維持すること
- RAW+JPEGペアでExif値を共有し、ファイルシステム値は共有しないこと
- チェック解除モードで非表示項目が未チェックになること
- チェック維持モードで表示外項目がコピー対象に残ること
- 全選択が表示中のコピー可能項目だけを変更すること
- 表示外チェック数とコピー総数が一致すること
- フィルタ解除時に外れたチェックを自動復元しないこと
- 再スキャンとコピー後の選択復元がフィルタ表示と独立して動作すること
